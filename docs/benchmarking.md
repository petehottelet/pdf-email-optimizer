# Benchmarking

Generate the redistributable fixtures, then run the harness:

```bash
python benchmarks/make_fixtures.py
python benchmarks/run_benchmarks.py --manifest benchmarks/benchmark_manifest.yaml --output benchmarks/results/latest.json
```

The harness writes:

- `benchmarks/results/latest.json`
- `benchmarks/results/latest.md`

For every successful case it records an honest original-vs-output render
comparison, so `worst_psnr`/`worst_rms` always reflect real pixel differences
(`inf`/`0.0` means a pixel-identical, lossless result). Missing fixtures are
reported as `skipped` and failures (e.g. encrypted input) as `failed`. Do not
replace skipped rows or hand-edit numbers.

## Gallery

`python benchmarks/make_gallery.py` renders before/after side-by-side images for
the small synthetic regression fixtures into `docs/gallery/` for the README.
Regenerate it whenever optimizer behavior changes.

## Demonstration brochures

The headline README gallery uses larger, realistic brochures (a real-estate
listing, a travel lookbook, and a restaurant menu) assembled from the curated,
synthetic stock images in `benchmarks/demo_assets/` (origins in that folder's
`PROVENANCE.md`):

```bash
python benchmarks/make_demo_brochures.py   # -> demo/originals/*.pdf (~10-14 MB, gitignored)
python benchmarks/make_demo_gallery.py     # optimize + render docs/gallery/*_listing.jpg etc.
```

`make_demo_brochures.py` embeds the placed photos losslessly so the source PDF is
honestly large, mimicking an Illustrator/InDesign/Canva/Keynote export. The large
source and optimized copies live under `demo/` and are not committed; only the
small before/after PNGs in `docs/gallery/` and the curated source images are.

## Fixture Rules

Only use PDFs that are:

- Generated by this project.
- Public domain.
- CC0.
- Explicitly redistributable.
- Small enough for normal repository use.

Do not commit commercial, confidential, client, or downloaded PDFs unless the license clearly allows redistribution.

## Required Case Types

The manifest reserves cases for photo-heavy brochures, deck exports, private creator payloads, screenshot-heavy reports, text/vector documents, scans, transparency, embedded metadata, repeated images, forms/annotations, and encrypted-file failure.
